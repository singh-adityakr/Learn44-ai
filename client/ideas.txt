Executive Context: The Imperative for Innovation in Motion
The global logistics ecosystem operates in a state of perpetual flux, a reality that project44
(p44) has mastered through its "Movement" platform, which functions as the connective
tissue for over 1.5 billion shipments annually.1 However, the internal velocity of a high-
growth technology enterprise often creates a paradoxical friction: as the complexity of the
product suite‚Äîspanning Transportation Management Systems (TMS), Yard Management,
and Ocean Visibility‚Äîexpands, the cognitive load placed on new team members increases
exponentially.2 The Q4 2025 "AI in Action" challenge, themed "Innovation in Motion,"
presents a unique opportunity to address this internal latency [User Query].
The proposed solution, Learn44.ai, is not merely a static knowledge base or a simple
chatbot. It is a comprehensive, agentic AI ecosystem designed to accelerate the "Time-to-
Productivity" for p44 employees, referred to internally as "Voyagers." By synthesizing the
company's vast repository of structured data (OpenAPI specs, EDI standards) and
unstructured knowledge (Confluence pages, Town Hall recordings, Slack threads),
Learn44.ai transforms the onboarding experience from a passive consumption of
documentation into an active, dialogue-driven engagement. This report outlines the
theoretical underpinnings, technical architecture, and strategic deployment plan for
Learn44.ai, positioning it as the definitive entry to secure victory in the Q4 challenge.
The strategic necessity of this tool is underscored by the diversity of the p44 product
portfolio. A new Solutions Engineer must navigate the nuances of "Port Intel" dwell time
data 2, understand the "Decision Intelligence" framework 3, and master internal
deployment tools like "Runway".4 Simultaneously, they must assimilate the cultural values
of "Real Talk" and "Sharing the Wheel".5 Learn44.ai acts as the unifying interface for these
disparate domains, leveraging Retrieval-Augmented Generation (RAG) and Generative
Video to deliver a "One Stop Solution" that is as dynamic as the supply chains p44
monitors.
The "Innovation in Motion" Alignment Strategy
The hackathon theme, "Innovation in Motion," calls for using AI to make work "smarter,
faster, and more meaningful" [User Query]. Learn44.ai directly addresses this triad:
‚Ä¢ Smarter: It replaces keyword searches with semantic understanding, allowing
users to ask complex, context-dependent questions about supply chain logic or
internal API structures.
‚Ä¢ Faster: It compresses hours of document hunting into seconds of retrieval,
automating the summarization of technical decks and codebases.
‚Ä¢ More Meaningful: By automating the repetitive "Level 1" support questions that
burden senior engineers (Inform Champions), it frees up human capital to focus on
high-impact innovation, allowing mentors to engage in deep architectural guidance
rather than syntax correction.

1. Domain Complexity and the "Voyager" Knowledge Gap
To construct an effective AI onboarding agent, one must first map the extensive domain
knowledge it must serve. The complexity of p44‚Äôs ecosystem creates a steep learning
curve for new hires, particularly those entering from outside the logistics sector.
Learn44.ai serves as a bridge across three primary "Knowledge Chasms."
1.1 The Language of Logistics (Supply Chain Fundamentals)
The first chasm is the specialized lexicon of the global supply chain. Software engineers
and product managers cannot effectively build or sell the "Movement" platform without a
nuanced understanding of the physical processes it digitizes. Learn44.ai incorporates a
Context-Aware Glossary Module that goes beyond dictionary definitions to explain terms
within the context of p44‚Äôs specific value proposition.
1.1.1 Detention, Demurrage, and Dwell Time
In the general logistics market, "Detention" and "Demurrage" are fee structures related to
container usage. However, within the p44 ecosystem, these terms are inextricably linked
to "Port Intel" and "Visibility" products.2
‚Ä¢ General Definition: Demurrage is the charge for a container remaining inside the
terminal beyond its free time; Detention is the charge for holding the container
outside the terminal.
‚Ä¢ p44 Context: Learn44.ai explains that p44 mitigates these costs using real-time
global container and vessel dwell data.
2 When a user asks, "How do we help
customers save money on ocean freight?", the AI connects the concept of
"Demurrage" directly to the "Port Intel" product features, explaining that accurate
ETAs allow shippers to retrieve containers before free time expires.
1.1.2 The Modal Spectrum (LTL, FTL, Ocean, Air)
The "Movement" platform aggregates data across all modes of transport.3 A new developer
must understand the structural differences between these modes to interact with the API
correctly.
‚Ä¢ LTL (Less-Than-Truckload): The AI explains that LTL involves hub-and-spoke
networks and "freight classes." It highlights that p44 recently introduced electronic
Bill of Lading (eBOL) capabilities to automate LTL workflows, which is critical for
anyone working on the "Documents" feature set.7
‚Ä¢ Ocean & Air: The system clarifies that tracking here often relies on Master Bill of
Lading (MBL) numbers or Container IDs, and that p44‚Äôs visibility extends to "Door-
to-Door" tracking for Air, including forwarder-managed segments.8

2.2 The Retrieval-Augmented Generation (RAG) Engine
The core intelligence of Learn44.ai is driven by a RAG pipeline that grounds the LLM in
project44‚Äôs proprietary data. This prevents the model from fabricating facts about the
supply chain or internal tools.24
2.2.1 Data Ingestion & Indexing Pipeline
The "One Stop Solution" requirement mandates the ingestion of diverse data sources. The
pipeline operates in four stages:
1. Extraction:
a. Confluence: Utilizes LangChain's ConfluenceLoader to scrape pages from
the "Engineering," "Product," and "HR" spaces. It filters for "active"
documents to avoid indexing legacy V1 API documentation.26
b. Developer Portal: Ingests OpenAPI (Swagger) specifications from
developers.project44.com (e.g., Movement API v4). This allows the AI to
understand the exact JSON schema required for endpoints like
/api/v4/shipments.13
c. Presentations: A custom python-pptx script extracts text and speaker notes
from Town Hall slides, capturing strategic directives that often don't make it
into formal documentation.14
2. Chunking:
a. 3. Embedding:
a. Documents are split into semantic chunks (e.g., 500 tokens with 50-token
overlap). Code snippets are chunked using CodeTextSplitter to preserve
function boundaries.15
Chunks are converted into vector embeddings using text-embedding-3-small
(or Azure equivalent).
4. Storage:
a. Vectors are stored in ChromaDB, tagged with metadata (e.g., source:
confluence, product: ocean, access_level: internal).
2.2.2 Hybrid Search Mechanism
To answer questions like "What is the error code 403 on the Runway deployment?", the
system uses a Hybrid Searchstrategy:
‚Ä¢ Keyword Search (BM25): essential for exact matches of unique internal acronyms
like "Launch44" or error codes.
‚Ä¢ Semantic Search (Vector): Essential for conceptual queries like "How do I fix
deployment failures?" where the user might not use the exact terminology found in
the docs.25
2.3 The Generative Video Agent (HeyGen Integration)
A standout feature for the "Innovation in Motion" challenge is the ability to generate
informational videos on the fly. This moves beyond text-based answers to multi-modal
teaching.
‚Ä¢ Workflow:
o User Query: "Teach me about the Connection Accelerator."
o LLM Scripting: The RAG engine retrieves the "Connection Accelerator" press
release and technical guide.9 It synthesizes a 150-word script summarizing
the key value proposition (e.g., "Instant connectivity to 80,000 carriers").
o API Dispatch: The system sends the script to the HeyGen API (POST
/v2/video/generate), selecting a "News Anchor" avatar to convey authority.22
o Delivery: The API returns a video stream URL, which is embedded in the
Streamlit UI, offering the user a dynamic visual briefing instead of a wall of
text.
